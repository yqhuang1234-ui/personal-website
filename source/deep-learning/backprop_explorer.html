The full backprop module has exactly three layers of functions, each calling the one below it. Understanding the call stack is half the battle.

Key insight: for every forward function there is a mirror backward function. Forward stores everything in a cache. Backward receives that cache and uses it to compute gradients — nothing is recomputed.

Example network used throughout

All concrete numbers use a 3-layer net: n⁰=2 inputs, n¹=3 hidden, n²=2 hidden, n³=1 output, m=2 examples.

Computing dW[l]

Why this formula? In forward: Z[l] = W[l] @ A[l-1] + b. So ∂Z/∂W = A[l-1]^T. Chain rule: dW = (1/m)·dZ @ A_prev.T. We divide by m to average over all examples.

Computing db[l]

Why sum over axis=1? b broadcasts across m examples, so its gradient is the sum of dZ across the example axis. keepdims=True keeps shape (n[l],1) not (n[l],).
Common mistake: forgetting keepdims=True gives shape (n[l],) instead of (n[l],1), causing shape mismatches in the update step.

Complete linear_backward()

def linear_backward(dZ, cache):
    A_prev, W, b = cache
    m = A_prev.shape[1]

    dW      = dZ @ A_prev.T / m
    db      = np.sum(dZ, axis=1, keepdims=True) / m
    dA_prev = W.T @ dZ

    return dA_prev, dW, db

linear_activation_backward wraps the activation step before calling linear_backward.

Step 1 — dZ via relu_backward

What relu_backward does: ReLU is g(Z)=max(0,Z). Its derivative is 1 where Z>0, 0 elsewhere. If a neuron was dead (Z≤0) in forward, its gradient is zero.

Step 1 — dZ via sigmoid_backward

What sigmoid_backward does: σ(Z) has derivative σ(Z)·(1−σ(Z)). We cached Z during forward so we recompute σ(Z) and multiply.

Step 2 — Call linear_backward (same for both)

def linear_activation_backward(dA, cache, activation):
    linear_cache, activation_cache = cache

    if activation == "relu":
        dZ = relu_backward(dA, activation_cache)
    elif activation == "sigmoid":
        dZ = sigmoid_backward(dA, activation_cache)

    dA_prev, dW, db = linear_backward(dZ, linear_cache)
    return dA_prev, dW, db

Full L_model_backward

def L_model_backward(AL, Y, caches):
    grads = {}
    L = len(caches)
    Y = Y.reshape(AL.shape)

    dAL = -(np.divide(Y, AL) - np.divide(1-Y, 1-AL))

    current_cache = caches[L-1]
    dA_prev, dW, db = linear_activation_backward(dAL, current_cache, "sigmoid")
    grads["dA"+str(L-1)] = dA_prev
    grads["dW"+str(L)]   = dW
    grads["db"+str(L)]   = db

    for l in reversed(range(L-1)):
        current_cache = caches[l]
        dA_prev, dW, db = linear_activation_backward(
            grads["dA"+str(l+1)], current_cache, "relu"
        )
        grads["dA"+str(l)]   = dA_prev
        grads["dW"+str(l+1)] = dW
        grads["db"+str(l+1)] = db

    return grads

Why l is always one less than the layer being processed: range(L-1) generates 0,1,…,L-2. At iteration l, caches[l] holds layer l+1's data.

Once you have all gradients, parameter update is simple: subtract a small step in the gradient direction.

Why subtract? Gradients point in the direction of steepest increase of J. We want to decrease J, so we subtract. α controls step size.

def update_parameters(params, grads, learning_rate):
    parameters = copy.deepcopy(params)
    L = len(parameters) // 2

    for l in range(L):
        parameters["W"+str(l+1)] -= learning_rate * grads["dW"+str(l+1)]
        parameters["b"+str(l+1)] -= learning_rate * grads["db"+str(l+1)]

    return parameters
