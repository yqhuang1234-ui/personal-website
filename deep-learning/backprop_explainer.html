Backpropagation

From the intuition of "blame assignment" to the full chain rule derivation — step by step, with live numbers and visualizations

The big idea: blame assignment
Backprop answers one question — which weights caused the error, and by how much?

Imagine you made a bad prediction. The loss is large. You need to update your weights. But there are hundreds of thousands of them — which ones are responsible?

Backpropagation is simply the systematic answer. It traces the error backwards through the network, assigning a "blame score" (gradient) to every single weight. The bigger the gradient, the more that weight contributed to the error, and the more we should adjust it.

Core intuition: The gradient dL/dW tells you — "if I increase W by a tiny amount ε, how much does the loss increase?" A large positive gradient means that weight is making things worse. We subtract it (gradient descent) to make things better.
The blue arrows show the forward pass — data flows left to right, building a prediction. The red arrows show backprop — the error gradient flows right to left, blaming each weight.

Think of it like a chain of dominoes. When the last domino (output) falls the wrong way, you trace back through each domino to figure out which earlier ones were misaligned. Each domino's contribution is its local derivative — the chain rule multiplies them all together.

1
Forward pass — make a prediction
Feed input X through the network layer by layer. Store every intermediate value Z and A — you'll need them in the backward pass.
2
Compute loss — measure the error
Compare prediction Ŷ to true label Y. Loss L is a single number measuring how wrong you are. Backprop's entire job is to reduce L.
3
Backward pass — trace the blame
Use the chain rule to compute dL/dW for every weight W. This tells each weight "how much did you contribute to the error?"
4
Gradient descent — fix the weights
Update: W := W − α · dL/dW. Subtract the blame. Repeat thousands of times until the loss is small.

Why not just try all weight combinations?

A network with 1 million parameters has 10^1,000,000 possible combinations to try. Backprop computes the exact direction to move every weight simultaneously in one backward pass. It's the only practical way to train deep networks.

What "gradient" means geometrically

The loss surface is a high-dimensional landscape. The gradient is a vector pointing uphill — in the direction of steepest increase. We subtract it to go downhill (minimize loss).

The learning rate α controls step size: too large → overshoot the valley, too small → takes forever.

Notation reminder

dL/dW = "derivative of Loss with respect to W"

In code we write dW as shorthand for dL/dW.

Same for dZ = dL/dZ, dA = dL/dA. The "dL/" part is always implied.

Key prerequisite: calculus chain rule

If L = f(g(x)), then dL/dx = (dL/dg) · (dg/dx).

Backprop is just this rule applied many times through many layers. Head to the Chain Rule tab to see it visually.

The chain rule — backprop's engine
Every gradient in backprop is a product of local derivatives chained together

Suppose you have a simple chain: x → z → a → L. Each arrow is a function. To find how x affects L, you multiply the local derivatives along the chain.

The key insight: each factor is a local derivative — it only involves variables directly connected to that node. Backprop computes them one at a time and multiplies them as it walks backwards. No single step requires knowing the whole network.

Let's work through each local derivative concretely:

Local derivative 1 — loss w.r.t. output a (for binary cross-entropy)
This is the "starting gradient" — how much the loss changes when the output changes. Large when prediction is wrong (far from y), near zero when correct.
Local derivative 2 — activation w.r.t. pre-activation z (for sigmoid)
This is the activation function's derivative. For ReLU it's 1 (z>0) or 0 (z≤0). For sigmoid it's σ(1−σ). This is why dying neurons have zero gradient — this factor kills the whole chain.
Local derivative 3 — pre-activation z w.r.t. weight w
Since z = wx + b, dz/dw = x. This is why inputs that are zero produce zero weight gradients — even if the error is large, the weight can't be blamed if its input was zero.
Putting it together: dL/dw = (dL/da) · (da/dz) · (dz/dw) = (loss sensitivity) × (activation slope) × (input value). Every backprop formula is just this pattern expanded to matrices.

Why transpose in the matrix version?

In the scalar case we just multiply derivatives. In the matrix case, the chain rule involves the Jacobian — a matrix of all partial derivatives. Working it out, the Jacobian of the linear transformation z = Wa is Wᵀ. So when we propagate gradients backward through a linear layer, we multiply by Wᵀ instead of W.

Matrix chain rule — propagating gradient through a linear layer

Worked scalar example

Let: w=0.5, x=2, b=0, y=1

Forward: z = 0.5×2 = 1.0
a = σ(1.0) = 0.731
L = −log(0.731) = 0.313

Backward:
dL/da = −1/0.731 = −1.368
da/dz = 0.731×(1−0.731) = 0.197
dz/dw = x = 2
dL/dw = −1.368 × 0.197 × 2 = −0.539

Update: w := 0.5 − 0.1×(−0.539) = 0.554

w increased because increasing w makes the prediction closer to 1 (=y).

Why does dL/dw = dZ/dw · dL/dZ?

Because z = wx+b, we have ∂z/∂w = x. And dL/dZ is the gradient coming back from the next step. Multiplying them gives: "how much does L change when w changes, via the path through z?"

Every weight gradient follows this same pattern — it's always (incoming gradient) × (local input).

Intuition for Wᵀ in backprop

Forward: z = Wa — W transforms the space from input-dim to output-dim.

Backward: we need to transform the gradient from output-dim back to input-dim. The inverse of a linear map uses the transpose: dA = Wᵀ · dZ.

Think of it as "undoing" the forward transformation to route the gradient back to where it came from.

Forward pass — building the computation graph
Every value you compute and store will be needed in the backward pass

The forward pass does two things: produces a prediction, and caches intermediate values (Z⁽ˡ⁾, A⁽ˡ⁾) that backprop will reuse. Think of it as leaving a trail of breadcrumbs.

StepFormulaWhat it computes & why
F1 Layer ℓ pre-activation Weighted sum. Each neuron takes a dot product of its weights with the previous layer's activations, plus a bias. Result Z⁽ˡ⁾ is (n⁽ˡ⁾, m) — one number per neuron per example.
F2 Layer ℓ activation Non-linearity. Applied element-wise — shape unchanged. Without this, stacking layers = one big linear layer = no expressive power.
F3 Output activation Sigmoid for binary classification. Squashes to (0,1) so output = probability. For multi-class, use softmax instead.
F4 Loss Binary cross-entropy. Measures how wrong the prediction is. −log(ŷ) when y=1 (want ŷ→1), −log(1−ŷ) when y=0 (want ŷ→0). Averaged over m examples.
Cache everything! During backprop, computing dW⁽ˡ⁾ requires A⁽ˡ⁻¹⁾ (stored from forward pass). Computing dZ⁽ˡ⁾ requires Z⁽ˡ⁾ (to evaluate g'). If you don't cache them, you'd have to recompute them — doubling your work.

Why store Z and A separately?

You need Z to compute g'(Z) in the backward pass (the activation derivative needs the pre-activation value).

You need A from layer ℓ−1 to compute dW⁽ˡ⁾ (it's the "input" to that layer's weights).

Storing both doubles memory usage but avoids expensive recomputation.

Why binary cross-entropy?

Cross-entropy comes from maximum likelihood estimation (MLE). If we assume outputs are Bernoulli probabilities, the log-likelihood of the training data under the model is exactly −J (the negative loss). Maximizing likelihood = minimizing cross-entropy loss.

It's also convex in the output (unlike squared error with sigmoid), making optimization well-behaved.

What if we used squared error instead?

With sigmoid + squared error, the loss surface becomes non-convex and gradients near saturation become extremely small (the sigmoid derivative multiplies in). Cross-entropy avoids this — its gradient ŷ − y is large even when the sigmoid is saturated.

Backward pass — tracing the gradient
The error flows right to left, each layer computing its own gradient using the chain rule

Backprop starts at the output and works backwards. At each layer it asks: "given the gradient coming from the right, what is the gradient for my weights, my bias, and what gradient do I pass to the left?"

The key backward pass pattern — every layer does exactly three things:

①
Compute dZ — pass gradient through activation
dZ⁽ˡ⁾ = dA⁽ˡ⁾ ∗ g'(Z⁽ˡ⁾)
Element-wise multiply (Hadamard product). The activation's local derivative g'(Z) "gates" how much of the incoming gradient passes through. A saturated sigmoid → g'≈0 → gradient killed.
②
Compute dW and db — gradients for this layer's parameters
dW⁽ˡ⁾ = (1/m) · dZ⁽ˡ⁾ · A⁽ˡ⁻¹⁾ᵀ
db⁽ˡ⁾ = (1/m) · sum(dZ⁽ˡ⁾, axis=1)
dW is the gradient used for gradient descent. The 1/m averages over all training examples.
③
Compute dA⁽ˡ⁻¹⁾ — gradient to pass to the previous layer
dA⁽ˡ⁻¹⁾ = W⁽ˡ⁾ᵀ · dZ⁽ˡ⁾
Wᵀ reverses the forward linear transformation. This is the "message" sent leftward. The previous layer receives it as its incoming gradient.
Why Wᵀ and not W? In the forward pass, W⁽ˡ⁾ has shape (n⁽ˡ⁾, n⁽ˡ⁻¹⁾). It maps from n⁽ˡ⁻¹⁾ dimensions to n⁽ˡ⁾ dimensions. In the backward pass, we need to map a gradient of shape (n⁽ˡ⁾, m) back to shape (n⁽ˡ⁻¹⁾, m). Transposing W gives (n⁽ˡ⁻¹⁾, n⁽ˡ⁾), and the multiply works: (n⁽ˡ⁻¹⁾, n⁽ˡ⁾)·(n⁽ˡ⁾, m) = (n⁽ˡ⁻¹⁾, m) ✓
Why dW = dZ · Aᵀ? From the chain rule: since Z = W·A + b, we have ∂Z/∂W = A (the input). The gradient w.r.t. W is (incoming dZ) combined with (the input A). Matrix calculus gives exactly dZ·Aᵀ — the transpose makes dimensions work out to match W's shape (n⁽ˡ⁾, n⁽ˡ⁻¹⁾).

Why * (Hadamard) for dZ but · (matmul) for dA?

dZ = dA ∗ g'(Z) uses element-wise multiply because g is applied element-wise — every neuron independently applies its activation. The chain rule for element-wise functions is element-wise.

dA⁽ˡ⁻¹⁾ = Wᵀ · dZ uses matrix multiply because Z = WA mixes neurons together — each element of Z depends on ALL elements of A through W. The chain rule for a linear mix is a matrix multiply.

Why 1/m for dW and db?

The cost function J is the average loss over m examples: J = (1/m)·ΣL⁽ⁱ⁾. Taking the derivative of an average introduces a 1/m factor. Without it, the gradient magnitude would depend on dataset size, making the learning rate meaningless.

Why sum for db but not dW?

The bias b has shape (n, 1) — one value per neuron. Its gradient over m examples must be summed (and divided by m) to collapse the m-column matrix to a single column. dW already has the right shape from dZ·Aᵀ, so no explicit sum is needed.

All 6 backprop equations
The complete set — with derivation reasoning for each one

For a 2-layer network (or any layer ℓ in a deeper network), backprop is described by exactly 6 equations. They always come in the same order.

#EquationWhy this formula?
1 dZ⁽ˡ⁾ Chain rule through the activation. dA⁽ˡ⁾ is the gradient coming in. g'(Z⁽ˡ⁾) is the local slope of the activation at each neuron. Element-wise multiply because g is applied element-wise. For the output layer with sigmoid + cross-entropy, this simplifies beautifully to just Ŷ − Y.
2 dW⁽ˡ⁾ Gradient of the loss w.r.t. weights. Since Z⁽ˡ⁾ = W⁽ˡ⁾·A⁽ˡ⁻¹⁾ + b, and dL/dW = dL/dZ · dZ/dW, and dZ/dW = A⁽ˡ⁻¹⁾ (the input), we get dZ · Aᵀ. Transpose makes the dimensions produce the same shape as W. Divide by m to average across examples.
3 db⁽ˡ⁾ Gradient of the loss w.r.t. bias. Since dZ/db = 1 (bias adds linearly), dL/db = dL/dZ · 1 = dZ. But dZ has m columns (one per example). Sum across examples (axis=1, keepdims=True) and divide by m. Result: same shape as b, which is (n⁽ˡ⁾, 1).
4 dA⁽ˡ⁻¹⁾ Gradient to pass to the previous layer. The Jacobian of Z = WA w.r.t. A is Wᵀ (from matrix calculus). So dL/dA = Wᵀ · dL/dZ. This is the gradient message sent left — the previous layer receives this as its incoming dA.
5 dA⁽ˡ⁾ output Special case: starting gradient at the output. For binary cross-entropy loss L = −[y·log(ŷ) + (1−y)·log(1−ŷ)], differentiating w.r.t. ŷ = A⁽L⁾ gives −y/ŷ + (1−y)/(1−ŷ). This is the gradient at the very start of backprop.
6 dZ⁽L⁾ simplified The beautiful simplification. When you combine equations 4 and 5 (dA⁽L⁾ × sigmoid derivative), the messy fractions cancel completely, leaving just Ŷ − Y. This is why the output layer's gradient is so simple — sigmoid + cross-entropy are mathematically matched to produce this clean result.
The simplification in equation 6: dL/dA × da/dZ = [−y/ŷ + (1−y)/(1−ŷ)] × ŷ(1−ŷ) = −y(1−ŷ) + (1−y)ŷ = ŷ − y. The sigmoid and cross-entropy are designed as a pair — together they produce the cleanest possible gradient.

Gradient descent update rule

After computing all gradients, update every parameter:

W⁽ˡ⁾ := W⁽ˡ⁾ − α · dW⁽ˡ⁾

b⁽ˡ⁾ := b⁽ˡ⁾ − α · db⁽ˡ⁾

α (learning rate) is a hyperparameter you choose. Typical values: 0.01 to 0.001.

Order matters — why right to left?

Computing dW⁽¹⁾ requires dZ⁽¹⁾. Computing dZ⁽¹⁾ requires dA⁽¹⁾. Computing dA⁽¹⁾ requires dZ⁽²⁾ from the next layer. So you must compute later layers first — hence right to left.

This is also why it's called backpropagation.

Generalization to L layers

For any L-layer network, apply equations 1–4 starting from layer L down to layer 1. At each layer, receive dA from the right, compute dZ, dW, db (for gradient descent), then compute dA to pass left.

The pattern is completely uniform — every hidden layer does the same four steps.

What if we use ReLU instead of sigmoid?

Equation 1 changes: dZ = dA ∗ (Z > 0). The ReLU derivative is 1 where Z>0 and 0 elsewhere. This acts as a binary mask — neurons that were active pass the gradient, dead neurons block it completely.

Equations 2, 3, 4 stay exactly the same.
